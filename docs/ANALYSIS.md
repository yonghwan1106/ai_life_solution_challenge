## 2. 대상 사용자 분석

### 2.1 주 사용자: 고령자 (60세 이상)

#### 인구통계 및 고충
- **73%**가 디지털 기기 사용에 어려움 경험
- **85%**가 키오스크 주문을 불편하거나 불가능하게 느낌
- 보이스피싱의 주요 피해자 (2024년 피해액 1.8조원)
- 시각/청각 저하로 식품 라벨 읽기 어려움
- 디지털 문해력 격차로 인한 사회적 고립

#### 핵심 니즈
1. 큰 텍스트/버튼이 있는 단순하고 직관적인 인터페이스
2. 읽기 부담을 줄이는 음성 기반 상호작용 (TTS/STT)
3. 금융 사기로부터의 실시간 보호
4. 복잡한 디지털 작업에 대한 단계별 안내
5. 안전망을 유지하면서 독립성 확보

### 2.2 부 사용자: 가족 보호자

#### 니즈
1. 고령 가족 구성원의 안전에 대한 안심
2. 침해적이지 않은 실시간 모니터링
3. 위험 상황에 대한 즉각적인 알림
4. 비정상적인 패턴을 감지하기 위한 활동 인사이트
5. 비상 연락처에 대한 빠른 접근

### 2.3 삼차 사용자: 시각/청각 장애인

#### 니즈
1. 시각 정보에 대한 오디오 설명 (제품 라벨, 화면)
2. 식품 안전을 위한 알레르기 유발 물질 경고
3. 시각 전용 시스템에 대한 대체 인터페이스

## 3. 구현 현황 분석

### 3.1 완료된 기능 ✅

#### 기능 1: 음성 바코드 리더
**상태**: ✅ 완전 구현됨

**기술 스택**:
- `html5-qrcode` + `@zxing/library` - 바코드 스캔
- Web Speech API (TTS) - 음성 출력
- `react-webcam` - 웹캠 통합

**구현 기능**:
- EAN-13, EAN-8, UPC-A, UPC-E, CODE 128, CODE 39, QR 코드 스캔
- 제품명, 제조사, 성분 읽기
- 알레르기 유발 물질 식별 및 경고 (밀, 계란, 우유, 대두 등)
- 음성 재생 "반복" 기능
- 스캔 프레임 오버레이를 통한 시각적 안내

**한계점**:
- 모의 제품 데이터베이스 사용 (2개 제품만 하드코딩)
- 식약처 API 연동 없음
- 제한적인 제품 커버리지

#### 기능 2: AI 키오스크 도우미
**상태**: ✅ 프로토타입 구현

**기술 스택**:
- `react-webcam` - 화면 캡처
- 모의 AI 분석 (화면 인식 시뮬레이션)
- Web Speech API - 음성 안내

**구현 기능**:
- 카메라 기반 화면 캡처
- 7단계 음성 지시
- 시각적 가이드 표시기
- 분석 이력 추적
- 실시간 모니터링 표시

**한계점**:
- 실제 컴퓨터 비전 AI 없음 (Google Cloud Vision API 미연동)
- 실제 화면 분석 대신 랜덤 단계 시뮬레이션
- 실제 키오스크 UI 요소 감지 불가
- 머신러닝 모델 없음

#### 기능 3: 보이스피싱 실시간 감지
**상태**: ✅ 기본 패턴 매칭 구현

**기술 스택**:
- Web Speech Recognition API (STT)
- 패턴 매칭 알고리즘 (키워드 감지)
- 실시간 전사 분석

**구현 기능**:
- 연속 오디오 모니터링
- 3단계 위험도 평가 (낮음/중간/높음)
- 실시간 키워드 감지:
  - **높은 위험**: 계좌번호, 송금, 금융감독원, 검찰청, 체포 (10개 키워드)
  - **중간 위험**: 대출, 저금리, 카드발급, 환불 (8개 키워드)
  - **낮은 위험**: 긴급, 즉시, 빨리 (5개 키워드)
- 즉각적인 음성 경고
- 보호자 알림 트리거
- 통화 이력 로깅
- 비상 연락처 바로가기 (112, 1332)

**한계점**:
- NLP/컨텍스트 분석 없음 (OpenAI GPT-4 미연동)
- 단순 키워드 매칭은 정교함 부족
- 톤, 말투, 사회공학 전술 감지 불가
- 실제 사기 데이터베이스 연동 없음

#### 기능 4: 보호자 대시보드
**상태**: ✅ UI 구현 (모의 데이터)

**기술 스택**:
- React 상태 관리
- 모의 사용자 데이터

**구현 기능**:
- 가족 구성원 상태 카드 (안전/경고/위험)
- 미확인 알림 카운터
- 심각도별 최근 알림 피드
- 주간 활동 통계 (바코드 스캔, 키오스크 사용, 피싱 차단)
- 활동 추세 차트 (7일 막대 그래프)
- 마지막 활동 타임스탬프
- 비상 연락처 빠른 다이얼
- 알림 확인 시스템

**한계점**:
- 실시간 데이터 동기화 없음
- 백엔드 데이터베이스 연결 없음 (PocketBase 설정되었으나 미연동)
- 푸시 알림 없음
- 다중 사용자 인증 없음
- 실시간 모니터링 대신 정적 모의 데이터

### 3.2 필수이나 누락/불완전 ❌

1. **데이터베이스 통합**
   - PocketBase 설정되었으나 프론트엔드 미연결
   - 7개 컬렉션 스키마 정의됨 (products, scan_history, voice_phishing_logs, kiosk_sessions, guardian_notifications, emergency_contacts, daily_activities)
   - 시드 데이터 스크립트 생성되었으나 기능은 모의 데이터 사용

2. **인증 시스템**
   - 로그인/회원가입 페이지 생성되었으나 기능하지 않음
   - 세션 관리 없음
   - 역할 기반 접근 제어 없음 (고령자 vs 보호자)
   - PocketBase 인증 미연동

3. **실제 AI/ML 모델**
   - 키오스크 화면 인식을 위한 Google Cloud Vision API 없음
   - 컨텍스트 피싱 분석을 위한 OpenAI GPT-4 없음
   - 온디바이스 처리를 위한 TensorFlow Lite 없음
   - 훈련된 커스텀 모델 없음

4. **공공 API 연동**
   - 식약처 식품 안전 데이터베이스 없음
   - 금융감독원 사기 데이터베이스 없음
   - 실시간 데이터 피드 없음

5. **알림 및 경보**
   - SMS 게이트웨이 없음
   - 푸시 알림 없음
   - 이메일 알림 없음
   - 보호자 알림은 UI 전용

6. **모바일 반응성**
   - 데스크톱 중심 디자인
   - 네이티브 모바일 앱 없음 (React Native 미구현)
   - 고령자를 위한 제한적인 터치 최적화

## 4. 기술 아키텍처 평가

### 4.1 강점 ✅

#### 프론트엔드 우수성
- **최신 스택**: Next.js 15 (App Router), TypeScript 5, Tailwind CSS 3
- **깔끔한 아키텍처**: 잘 조직된 파일 구조
  ```
  app/
    ├── barcode/page.tsx
    ├── kiosk/page.tsx
    ├── voicephishing/page.tsx
    ├── dashboard/page.tsx
    ├── login/page.tsx
    ├── signup/page.tsx
    └── about/page.tsx
  lib/
    ├── utils.ts (TTS, 패턴 매칭)
    └── pocketbase.ts (DB 클라이언트)
  components/
    ├── features/
    └── ui/
  ```
- **타입 안전성**: 포괄적인 TypeScript 인터페이스
- **코드 품질**: 깔끔하고 읽기 쉬우며 잘 주석 처리됨
- **반응형 디자인**: 현대적인 애니메이션이 있는 Tailwind 유틸리티

#### 브라우저 API 활용
- **Web Speech API**: 백엔드 의존성 없이 TTS/STT 우수하게 사용
- **Webcam API**: 적절한 `react-webcam` 통합
- **MediaDevices**: 견고한 카메라 권한 처리

#### UX 디자인
- **시각적 계층**: 명확한 아이코노그래피가 있는 크고 화려한 카드
- **애니메이션**: 부드러운 전환, 그라데이션 배경, 플로팅 요소
- **접근성 고려사항**:
  - 높은 대비 색상 구성
  - 큰 터치 타겟
  - 음성 우선 상호작용
  - 문제 해결 팁이 포함된 명확한 오류 메시지

### 4.2 약점 ⚠️

#### 1. 백엔드 통합 부족
- 프론트엔드 전용 구현
- 영구 데이터 저장소 없음
- 교차 기기 동기화 없음
- 실제 데이터베이스 쿼리 대신 모의 데이터

#### 2. 실제 AI 모델 없음
- 패턴 매칭은 "AI"가 아님 - 단순 문자열 매칭
- 머신러닝, 모델 훈련, 컴퓨터 비전 없음
- "AI 역량"에 대한 오해의 소지가 있는 주장

#### 3. 보안 우려
- 인증/권한 부여 없음
- .env에 하드코딩된 API 키 (사용 시)
- 데이터 암호화 없음
- HTTPS 강제 없음
- 입력 유효성 검사/살균 없음

#### 4. 확장성 문제
- 모의 데이터 구조는 확장되지 않음
- 캐싱 전략 없음
- 정적 자산을 위한 CDN 없음
- 데이터베이스 인덱싱 없음
- 속도 제한 없음

#### 5. 테스트 격차
- 단위 테스트 없음
- 통합 테스트 없음
- E2E 테스트 없음
- 접근성 테스트 없음
- 성능 벤치마크 없음

## 5. 고령자 UX/UI 적합성

### 5.1 우수한 디자인 선택 ✅

1. **크고 화려한 인터페이스 요소**
   - 80px 이상 아이콘 크기
   - 높은 대비 그라데이션 (녹색, 파란색, 빨간색, 보라색)
   - 명확한 시각적 계층
   - 기능 카드를 위한 번호 배지 (1, 2, 3, 4)

2. **음성 우선 상호작용**
   - 모든 주요 작업에 음성 피드백
   - 전체적인 "반복" 버튼
   - 0.9배속 오디오 지침 (약간 느림)
   - 한국어 (ko-KR) TTS

3. **명확한 지침**
   - 단계별 번호 목록
   - 문제 해결 팁이 있는 정보 상자
   - 시각적 가이드 (녹색 스캔 프레임, 손 포인터 이모지)
   - 실시간 상태 표시기 ("감지 중", "스캔 시도 중: X회")

4. **오류 처리**
   - 친근한 오류 메시지
   - 실행 가능한 솔루션 (예: "주소창 옆의 자물쇠 아이콘을 클릭하세요")
   - 여러 재시도 기회

5. **인지 부하 감소**
   - 단일 목적 화면 (페이지당 하나의 기능)
   - 최소 탐색
   - 영구 "홈으로" 버튼
   - 복잡한 메뉴나 중첩 옵션 없음

### 5.2 개선 영역 ⚠️

1. **폰트 크기 사용자 정의**
   - 사용자 조정 가능한 텍스트 크기 없음
   - 현재 크기는 적절하지만 사용자 정의 불가
   - 고대비 모드 토글 없음

2. **터치 타겟**
   - 일부 버튼 < 44px (iOS 최소 권장)
   - 모바일 기기의 가까운 간격
   - 햅틱 피드백 없음

3. **온보딩**
   - 처음 사용자 튜토리얼 없음
   - 카메라 권한 지식 가정
   - 연습 모드 또는 데모 흐름 없음

4. **언어 복잡성**
   - 일부 기술 용어 ("패턴 매칭", "정확도 95%")
   - 단어를 더 단순화할 수 있음
   - 더 간단한 언어 모드 옵션 없음

5. **탐색**
   - 브라우저 뒤로 버튼에 의존
   - 영구 탐색 메뉴 없음
   - 기능 간 이동 어려움

6. **피드백 메커니즘**
   - 햅틱/진동 알림 없음
   - 시각 + 오디오에만 의존
   - 긴급성을 위한 색상 코딩 추가 가능
